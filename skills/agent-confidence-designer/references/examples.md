# 示例模式

## Excel 销售报表 Agent

结果单元：

- 合并后的明细；
- 月度销售总额；
- 区域汇总；
- 报告中的事实数字；
- 图表和格式。

典型风险：

- 混合日期格式导致月份归类错误；
- 隐藏行被漏处理；
- 重复文件被重复导入；
- 汇总公式引用范围错误；
- Word 报告数字与 Excel 不一致。

主要确定性验证：

- 输入文件清单；
- 日期可解析率；
- 记录数守恒；
- 主键唯一；
- 独立复算与汇总勾稽；
- 跨文件数字一致性。

关键月度总额只有在所有必要验证通过时才允许为高。

## Word 经营分析 Agent

结果单元：

- 文档结构完整性；
- 报告中的事实数字；
- 趋势描述；
- 原因分析；
- 行动建议。

典型风险：

- 漏掉固定章节；
- 报告数字与数据源不一致；
- 时间口径错位；
- 把相关性写成因果性；
- 原因分析没有数据支持。

主要验证：

- 章节清单与模板检查；
- 数字、指标、期间和来源交叉核对；
- 结论到数据证据的映射；
- 对因果表述使用语义辅助检查和人工门禁。

对于缺少稳定自动验证的原因分析，即使没有发现明显问题，也通常最高为中，除非业务提供了明确、可验证的证据规则。
