# 三个实际案例

## 1. PBA SO 合同转换

最小结果可以是合同字段或任务行。可信含义重点是字段来源、映射规则和回退路径可靠性。高可信不证明源 SO 本身业务录入绝对正确。

建议维度：直接来源、规则命中、回退方式、来源追溯、字段间一致性。中低结果应定位到具体单元格、源 sheet、源行和规则。

## 2. ECS UAT 自动化

至少拆分规则理解、单条测试数据和测试集合三个层次。固定支持规则完整命中可以支持高；部分匹配、组合推导或强行生成应降低等级。

集合型结果必须定义 clean、dirty、上下边界、缺失或特殊值等必要维度。只比较数量不足以判断完整性。

## 3. 供应商调查问卷

最小结果是每一道问题、答案、选项或附件要求。文本相似度只能说明历史上出现过类似问题，不能证明答案适用于当前法人主体、时间、客户和用途。

建议维度：语义匹配、主体一致性、来源权威性、时效性、多来源一致性和答案完整性。主体不一致、高影响问题无责任部门证据、仅由模型生成等条件应作为硬性规则。
