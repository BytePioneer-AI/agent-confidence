# 示例索引

## Excel 销售汇总

读取 [Excel 销售汇总完整示例](examples/excel-sales-report.md)。

重点检查：

- 已有实现时，先写当前实际结果溯源链，再确认结果单元和设计高、中、低；
- 只把业务需要使用和验收的指标或内容定义为结果单元；
- 将内部标准化明细作为结果溯源，除非它本身也是明确业务输出；
- 为每个汇总结果关联实际来源记录、计算口径和结果位置；
- 不为完整 Excel、Word 或整个任务定义总体置信度。

## Word 经营报告

读取 [Word 经营报告完整示例](examples/word-management-report.md)。

重点检查：

- 已有实现时，先写当前实际结果溯源链，说明每类内容实际上使用了哪些来源；
- 按用户可独立确认或修正的事实、趋势、原因和建议拆分结果；
- 为每个结果关联实际数据、文档位置或业务规则；
- 不把报告结构、生成步骤或完整 Word 当作置信度对象；
- 对事实错误和“现有来源是否足以支持这句话”分别定义结果级条件和具体检查方式。

## 使用示例

不要照搬结果类型或高、中、低条件。始终根据当前业务确认：

- 什么是业务明确输出；
- 用户按什么单位接受、退回或修正；
- 已有实现时，当前结果实际使用了哪些来源、怎样形成；
- 每个结果必须记录和展示哪些溯源信息；
- 什么条件允许直接使用；
- 什么条件需要检查指定内容、修正或阻断。

尚无业务实现时，不生成当前实际链，也不根据理想流程猜测；只设计结果单元、必须保留的溯源信息和高、中、低条件。
