# 结果置信度条件参考

仅在用户提供真实测试问题、用户反馈，或需要检查置信度条件覆盖面时使用。不要根据本清单虚构业务问题，也不要单独生成“已知问题与风险”章节。

发现问题后，将其分析为可能影响某类业务结果的条件；只有用户确认后，才修改当前业务置信度定义。

## 来源与适用性

- 结果没有可定位来源；
- 使用了错误来源或旧版本来源；
- 来源主体、年份、范围或业务口径不适用于当前结果；
- 多个实际来源相互冲突；
- 结果遗漏了实际使用的来源。

## 映射与转换

- 来源字段与结果字段映射错误；
- 日期、币种、单位或格式转换错误；
- 筛选、去重、合并或分类规则应用错误；
- 结果写入了错误的业务位置；
- 来源—结果形成关系无法说明。

## 计算与汇总

- 公式范围或输入记录错误；
- 聚合口径、时间窗口、币种或精度错误；
- 结果无法由所列来源和规则复算；
- 多个关联结果之间发生事实冲突。

## 内容与语义

- 结果与来源事实矛盾；
- 结果遗漏来源中的关键限制；
- 将间接迹象写成确定事实或因果结论；
- 结论或建议缺少实际来源支持；
- 表述强度超出来源能够支持的程度。

## 支持范围

- 输入或业务场景超出已确认支持范围；
- 权威数据、审批或必要业务判断缺失；
- 结果依赖尚未确认的业务假设；
- 当前规则无法稳定识别该类结果。

## 分析真实问题

对真实问题至少记录：

```text
原始描述
+ 发生场景
+ 可能影响的结果单元
+ 建议修改的置信度条件
+ 建议审查动作
+ 用户决定
+ 最终处理
```

不要为来源、处理节点或问题本身定义高、中、低。不要使用通用严重度等级代替具体业务后果。
