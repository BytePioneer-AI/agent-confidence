# Excel 销售报表 Agent 可信等级设计审查

## 1. 结论

- 设计成熟度：可试点
- 是否具备上线条件：示例验证器与问题覆盖已完整；真实业务接入时仍需使用生产数据回归
- 关键原则：月度销售总额只有在明细守恒、日期标准化、独立复算和来源追溯均通过时才可为高

## 2. 结果覆盖

三个 high/critical 结果均配置确定性必要验证器。输出文件完整性属于低关键结果，不影响核心数值自动放行。

## 3. 已知问题覆盖

混合日期、隐藏行业务记录遗漏和 Word/Excel 数字不一致均绑定确定性验证器和回归测试。

## 4. 人工审查动作

- 日期异常：检查具体 record_id 和原始日期值；
- 记录数不守恒：检查隐藏行、过滤行和重复文件；
- 汇总不一致：检查过滤条件、单位和公式范围；
- 跨文件不一致：核对指标值、期间和币种。
